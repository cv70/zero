# zero вҖ” Universal Agent Runtime Platform

> **з®ҖдҪ“дёӯж–Ү**: [README.zh-CN.md](./README.zh-CN.md)

## Overview

**zero** is a **universal Agent runtime platform** built with Rust, designed to be:

- **Universal** вҖ” Works for developers, enterprises, and individual users
- **Rust-first** вҖ” High performance, low resource footprint (<5MB RAM)
- **Trait-driven** вҖ” Everything is swappable through Trait definitions
- **Extensible** вҖ” Core stays minimal, capabilities come from extensions

## Core Features

### Trait-Driven Architecture

All core capabilities are defined as Rust Traits:

| Trait | Purpose |
|-------|---------|
| `Agent` | Agent factory and execution |
| `Tool` | Unified tool abstraction |
| `GlobalSharedMemory` | Cross-agent memory |
| `LLMProvider` | Model provider abstraction |
| `Channel` | Message channel abstraction |

### Unified Tool System

- **Declarative Tools** вҖ” JSON/YAML definitions
- **MCP Integration** вҖ” Model Context Protocol support
- **Skills** вҖ” File-based skill definitions
- **Rust Implementation** вҖ” Direct Trait implementation

### Multi-Model Support

```rust
pub enum ModelCapability {
    TextOnly,          // Text only
    TextAndImages,     // Text + Images
    TextAndVideo,      // Text + Video
    Multimodal,        // Text + Images + Video + Audio
}
```

### Layered Memory System

- **Agent Isolation** вҖ” Each agent has independent context
- **Global Shared Memory** вҖ” Cross-agent knowledge sharing via Trait
- **Filesystem Storage** вҖ” Default implementation using filesystem structure

## Quick Start

### Installation

```bash
# Build from source
cargo build -p zero-cli

# Run CLI
cargo run -p zero-cli -- --help
```

### Basic Usage

```bash
# Start interactive session
zero run

# Execute single command
zero exec "Hello, world!"

# List available agents
zero list-agents

# List available tools
zero list-tools
```

### Configure Models

```bash
# Add model providers
zero provider add openai --api-key $OPENAI_API_KEY
zero provider add anthropic --api-key $ANTHROPIC_API_KEY
zero provider add ollama --endpoint http://localhost:11434
```

## Design Principles

1. **YAGNI** вҖ” Remove all unnecessary features
2. **Trait-driven** вҖ” Everything is swappable
3. **Extension-first** вҖ” Core stays minimal
4. **Unified abstraction** вҖ” Eliminate differences through Traits
